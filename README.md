# Portfolio - Marian Wachała

Personal portfolio website showcasing my software development projects and skills.

## Setup

### Opening the Portfolio Locally

1. **Clone the repository:**
```bash
git clone https://github.com/Maniekw12/Projekt1-TechnikiWWW.git
cd Projekt1-TechnikiWWW
```

2. **Open in browser:**
   Simply open `index.html` in your web browser.


## User Stories - Portfolio Website

### 1. As a recruiter, I want to quickly see the most important information about the author, so that I can evaluate his profile and competencies.


### 2. As a potential employer, I want to easily browse the author's projects, so that I can assess his skills and portfolio.

### 3. As a user, I want to navigate to a project page by clicking its card, so that I can quickly access its details.

### 4. As a recruiter, I want access to the author's LinkedIn profile, so that I can verify his professional background.


### 5. As a user, I want to open the “About” modal, so that I can read additional information about the author.

### 6. As a user, I want the website to be responsive, so that it works properly on mobile, tablet, and desktop devices.


### 7. As a recruiter, I want to see the list of technologies the author uses, so that I can understand his technical strengths.

### 8. As a user, I want to easily contact the author, so that I can reach him for collaboration or job opportunities.


### 9. As a visitor, I want to view screenshots of a project, so that I can understand how the applications look and work.


### 10. As a user, I want to easily return to the homepage from any project page, so that I can continue browsing the portfolio.
